// ABOUT THE PROJECT

What is VigilAI?

VigilAI is an autonomous AI-powered moderation system designed for live Twitch streams. It monitors chat messages in real-time, detects toxic behavior using a cascading AI pipeline, and takes automated moderation actions — all running 100% locally on your machine with zero API costs.

Unlike cloud-based moderation services, VigilAI runs entirely on-device. Your chat data never leaves your computer. It's designed to be deployed on a secondary machine (like a Mini PC) next to the streamer's main rig, acting as a silent, intelligent guardian of the community.

// ARCHITECTURE

The Hybrid Traffic Light System

VigilAI uses a multi-layered AI pipeline inspired by a traffic light metaphor. Each layer acts as a filter, escalating only when necessary:

01

🟢 GREEN FILTER — Whitelist

VIPs, Moderators, and Subscribers pass through directly. Trusted users are never flagged. Cost: $0.

02

🔴 RED FILTER — BERT (Local)

Every message is scored by a local BERT model for toxicity. Messages scoring above 95% are immediately actioned. Runs in ~5ms. Cost: $0.

03

🟡 YELLOW FILTER — LLM (Ollama)

Ambiguous messages (BERT 70–95%) are sent to a local LLM (Gemma 4 31B) for contextual analysis. It understands sarcasm, slang, and cultural nuance. Cost: $0.

04

🔵 BLUE FILTER — Personality

The same LLM generates witty, in-character responses to toxic users before timing them out. It's not just a mod — it's a personality. Cost: $0.

// EDGE DEPLOYMENT

Self-Hosted on a GMKtec Mini PC — Full MLOps Pipeline

VigilAI runs as an always-on edge service on a dedicated GMKtec Mini PC with Ubuntu Server 24.04 LTS. The entire ML inference stack — BERT toxicity classification and Gemma 4 31B via Ollama — executes locally on-device, ensuring zero latency, full data privacy, and $0 recurring costs.

The deployment uses a Docker Compose orchestration for the bot container while Ollama runs natively on the host for direct hardware access. Updates are deployed via a git-based CI/CD script, and the web console is securely exposed through Cloudflare Tunnel — no open router ports, no Nginx needed.

01

🖥️ OS & Hardware

GMKtec Mini PC (16GB+ RAM, Intel N100/AMD Ryzen). Fresh install of Ubuntu Server 24.04 LTS. Headless — managed entirely via SSH. Silent, low-power, always-on.

02

🧠 Ollama (Native)

Ollama is installed directly on the host OS (not in Docker) for maximum performance and direct CPU/RAM access. The Gemma 4 31B model is pulled and served on localhost:11434. Runs as a systemd service with auto-start.

03

🐳 Docker Compose

The VigilAI bot runs inside a Docker container orchestrated by docker-compose.yml. Uses network_mode: host to reach Ollama on localhost. Configured with restart: unless-stopped, log rotation, and memory limits.

04

🔒 Cloudflare Tunnel

The web console is exposed to the internet via Cloudflare Tunnel (cloudflared). No open router ports, no Nginx, no SSL cert management. Encrypted end-to-end with zero-config DNS pointing to localhost:8000.

05

🔄 CI/CD Deploy Script

One-command deployments via scripts/deploy.sh: pulls latest code from GitHub, rebuilds the Docker image, and restarts the service. No downtime, no manual steps. Future: GitHub Actions webhook for fully automated pushes.

06

🚀 Boot Sequence

Power on → Ubuntu boots → systemd starts Ollama → Gemma 4 model loads into RAM → Docker Compose starts VigilAI container → Cloudflare Tunnel exposes console → fully autonomous, zero human intervention.
